6 ‚Äî Design Principles

Terminal-grade clarity

Every element earns its space. No decoration for decoration's sake. Agent output is shown in dark code blocks that feel native to developers. Status is communicated through color-coded badges, not verbose labels.

Email metaphor, dev execution

The compose ‚Üí send ‚Üí thread flow feels familiar like email, but recipients can be machines. To/CC semantics apply to both humans and agents. The inbox is the single source of truth for all team activity.

Agents are first-class citizens

Agents get their own avatar style (bordered + icon), status indicators, and identity in threads. They're visually distinct from users but treated as equals in conversation. Active/inactive state is always visible.

Dark sidebar, light workspace

Navigation lives in a dark sidebar that anchors the interface. The main workspace is white and spacious for reading. Code output uses the same dark palette as the sidebar, creating visual consistency.
